# 借助Blazor在浏览器运行dotnet代码

**本文重点**，想看步骤的请直接移步[正文](#正文开始)

- 如何精简`Blazor`的代码，移除界面部分
- 如何和`c#`部分交互
- 如何访问`FS`部分
- 打包构建

为何要做这么麻烦的事情？起因是工作上遇到一个需求，需要在前端解码`usm`格式的视频, 因为没有公开编码格式，所以没办法像之前写`vmd.js`一样直接去解析二进制流了

但是第三方好用的通用的只有`vgmtoolbox`这个`c#`应用，又不能用在浏览器上，如果是`c`写的倒是可以用`emsdk`编译成`asm`来在浏览器跑

也就差不多只有两条路可以走了

- 想办法编译`c#`
- 自己重写成`typescript`

其实我有尝试过编译成`typescript`, 可惜，`netjs`缺了很多`mslib`需要的库，而且有些部分转完之后很奇怪，如果自己重头开始写？。。。毅力真的不够

好在，发现微软大佬出了一款`Blazor`，是用`c#`来构建`spa`的，可以利用一下

## Blazor

其实在使用`Blazor`之前，发现`mono`有一个`mono-wasm`的项目，但是好死不死的编译器网站说是最近在迁移，无法下载了，而且`mono-wasm`已经被`Blazor`整合进去了，所以直接用`Blazor`即可

### Blazor的几个问题

一直没下定决定用`Blazor`的一个主要原因是，`Blazor`是用来构建页面的，但是我只是需要利用它的接口能力而已，这方面也没人试过如何删掉界面部分

另一个原因是，其实这块工作扔给后端转码比较好，`webassembly`其实也挺慢的

## 正文开始

### 创建BlazorApp

首先，你得安装`dotnet`, 请安装`dotnet5`

安装完成后，命令行内应该就能直接访问`dotnet`了

接着运行

```bash
dotnet new blazorwasm -o BlazorApp --no-https
```

创建`BlazorApp`当然你也可以改成其他名字
